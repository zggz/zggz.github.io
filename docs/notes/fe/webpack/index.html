    extensions妙用 #

    在项目的开发过程中通常需要根据不同的参数加载不同的页面文件，通过webpack的extensions可以剥离业务代码耦合的情况下，分别加载。

    实例 #

    1. 入口文件app.js
      1import Home from "./Home"
2function App() {
3return (
4    <div className="App">
5        <Home/>
6    </div>
7);
8}
9export default App
    2. 需要加载的文件
    • Home/index.js
      1function Home() {
2return <h1>TT Home</h1>
3}
4export default Home
    • Home/index.tt.js
      1function Home() {
2return <h1>TT Home</h1>
3}
4export default Home
    1. 在webpack中配置
      • process.env.TYPE 可以通过cross-env 注入
      1resolve{
2    extensions:[
3        // 这里需要注意优先级需要写在前面
4        `.${process.env.TYPE}.js`,
5        '.js'
6    ]
7}
    2. 现在你可以在通过不同的type参数来控制程序打包时候加载的页面，并且对业务代码是无入侵的
